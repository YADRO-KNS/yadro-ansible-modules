yadro.obmc.bmc_restart module – Restart BMC.

Note

This module is part of the yadro.obmc collection (version 1.1.0).

You might already have this collection installed if you are using the ansible package. It is not included in ansible-core. To check whether it is installed, run ansible-galaxy collection list.

To install it, use: ansible-galaxy collection install yadro.obmc.

To use it in a playbook, specify: yadro.obmc.bmc_restart.

New in version 1.0.0: of yadro.obmc

Synopsis

  • This module supports check mode.

Parameters

Parameter

Comments

connection

dictionary / required

connection describes OpenBmc connection configuration. Two authentication methods

available (username and password or session_key). Session key can be received using

bmc_session module. One of authentication methods must be used.

hostname

string / required

BMC server IP address or hostname.

password

string

BMC user password.

port

integer

BMC REST API port.

Default: 443

session_key

string

BMC session key.

timeout

integer

BMC REST API request timeout.

Default: 30

username

string

BMC username to login.

validate_certs

boolean

Responsible for SSL certificates validation.

If set to False certificates won’t validated.

Choices:

  • no

  • yes ← (default)

force

boolean

It indicates BMC will be restart right away. May cause data corruption.

Choices:

  • no ← (default)

  • yes

Examples

---
- name: Restart BMC
  yadro.obmc.bmc_restart:
    connection:
      hostname: "localhost"
      username: "username"
      password: "password"

- name: Force restart BMC
  yadro.obmc.bmc_restart:
    connection:
      hostname: "localhost"
      username: "username"
      password: "password"
    force: true
